# MCP Stack — LM Studio Configuration

> This page documents the exact changes made to get `docker compose up` working with a local **LM Studio** backend and the `{service}bdBD1974` password convention.

## What Was Configured

The `docker-compose.yml` already had all passwords baked in as defaults — no changes to that file were needed.
All configuration was done by creating the **`.env`** file at:

```
C:\Users\swhyo\Documents\Logseq\mcp-stack\.env
```

---

## The `.env` File (full content)

```dotenv
###############################################################################
# MCP Stack — .env
# NEVER commit this file to version control.
###############################################################################

# ── Logseq ───────────────────────────────────────────────────────────────────
# !! ACTION REQUIRED !!
# Generate this token inside Logseq:
#   Settings → Features → HTTP API → enable → copy one of the listed tokens
LOGSEQ_API_TOKEN=REPLACE_WITH_YOUR_LOGSEQ_TOKEN
LOGSEQ_API_URL=http://host.docker.internal:12315

# ── Firecrawl ─────────────────────────────────────────────────────────────────
# Default key bundled in docker-compose works for quick testing.
# Replace with your own at: https://www.firecrawl.dev/app/api-keys
FIRECRAWL_API_KEY=fc-3c2677e6221e4893aff7f02eb5886b45

# ── Elasticsearch ─────────────────────────────────────────────────────────────
ELASTIC_USERNAME=elastic
ELASTIC_PASSWORD=elasticbdBD1974

# ── SearXNG ───────────────────────────────────────────────────────────────────
SEARXNG_SECRET=searxngbdBD1974

# ── Mem0 — LM Studio (fully local) ───────────────────────────────────────────
MEM0_LLM_PROVIDER=openai
MEM0_LLM_BASE_URL=http://host.docker.internal:1234/v1
MEM0_LLM_API_KEY=lm-studio
MEM0_LLM_MODEL=google/gemma-4-26b-a4b-qat

MEM0_EMBEDDER_PROVIDER=openai
MEM0_EMBEDDER_BASE_URL=http://host.docker.internal:1234/v1
MEM0_EMBEDDER_API_KEY=lm-studio
MEM0_EMBEDDER_MODEL=text-embedding-mxbai-embed-large-v1
MEM0_EMBEDDER_DIMS=1024
```

## Password Convention Applied

All **internal** service credentials follow the `{service}bdBD1974` pattern:

| Service | Variable | Value |
|---------|----------|-------|
| Elasticsearch | `ELASTIC_PASSWORD` | `elasticbdBD1974` |
| SearXNG | `SEARXNG_SECRET` | `searxngbdBD1974` |

> **Note on other credentials:**
> - `LOGSEQ_API_TOKEN` — auto-generated by Logseq (not a password we set). See action below.
> - `FIRECRAWL_API_KEY` — external API key from firecrawl.dev; uses default bundled in compose.
> - `MEM0_LLM_API_KEY` / `MEM0_EMBEDDER_API_KEY` — LM Studio ignores these; `lm-studio` is the standard placeholder.

---

## LM Studio Models Mapped

| Role | LM Studio Model Identifier | Variable |
|------|---------------------------|----------|
| Agent / Chat | `google/gemma-4-26b-a4b-qat` | `MEM0_LLM_MODEL` |
| Embedder | `text-embedding-mxbai-embed-large-v1` | `MEM0_EMBEDDER_MODEL` |

**Why `MEM0_EMBEDDER_DIMS=1024`?**
The `mxbai-embed-large-v1` model from Mixedbread AI outputs **1024-dimensional** vectors.
The `docker-compose.yml` default was `768` — this `.env` overrides it to `1024` so Elasticsearch creates the correct index mapping.

**Why `MEM0_LLM_PROVIDER=openai`?**
LM Studio exposes an OpenAI-compatible REST API at `http://localhost:1234/v1`.
Setting the provider to `openai` makes Mem0 use that API format. `host.docker.internal` is Docker Desktop's magic hostname that resolves to the Windows host machine from inside any container.

## !! One-Time Action Required: Logseq API Token

The `logseq-mcp` container cannot start without a real Logseq token:

1. Open **Logseq** on your desktop
2. Go to `Settings → Features → HTTP API`
3. Toggle **HTTP API server** ON — Logseq restarts its API on port `12315`
4. Click **Add token**, copy the generated token
5. Open `.env` and replace `REPLACE_WITH_YOUR_LOGSEQ_TOKEN` with your token

## Troubleshooting

- **Elasticsearch won't start** → Run once in elevated PowerShell:
  ```powershell
  wsl -d docker-desktop sysctl -w vm.max_map_count=262144
  ```
- **Mem0 embedding errors** → Verify LM Studio has `text-embedding-mxbai-embed-large-v1` loaded and the server is running on port `1234`
- **Mem0 LLM errors** → Verify `google/gemma-4-26b-a4b-qat` is loaded in LM Studio
- **Logseq MCP fails** → Check your `LOGSEQ_API_TOKEN` in `.env` and confirm Logseq's HTTP API is enabled
